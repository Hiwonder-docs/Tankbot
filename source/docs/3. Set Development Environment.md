# 3.Set Development Environment

> [!NOTE]
>
> To avoid compilation failure and influence on normal use, please activate Keil software after installation.

For activation method, you can refer to the file “Keil Activation Tutorial” ”saved in “Keil Installation Package and mcuisp Dowdloader”. If you use Keil for commercial purpose, please purchase legitimate version.

## 3.1 Keil software installation

Keil C51 is a powerful software customized for microcontroller.

1)  Please double click to open “mdk500.exe” file in “Keil Installation Package and mcuisp Dowdloader” folder.

2)  Tick “I agree”.

<img class="common_img" src="../_static/media/chapter_3/section_1/media/image1.png" style="width:800px" />

3)  Select the folder where Keil will be installed.

<img class="common_img" src="../_static/media/chapter_3/section_1/media/image2.png" style="width:800px" />

4)  You will be required to enter the user information, and you can fill in at will.

<img class="common_img" src="../_static/media/chapter_3/section_1/media/image3.png" style="width:800px" />

5)  After entering, click “Next” to install.

<img class="common_img" src="../_static/media/chapter_3/section_1/media/image4.png" style="width:800px" />

6)  Please wait for a while, a window will pop up asking you to install this software, and just click “Install”

<img class="common_img" src="../_static/media/chapter_3/section_1/media/image5.png" style="width:800px" />

7)  Click “Finish” after installation.

<img class="common_img" src="../_static/media/chapter_3/section_1/media/image6.png" style="width:800px" />

## 3.3 Methods to download program

<img class="inline-icon" src="../_static/media/chapter_3/section_3/media/image1.png" style="width:30px" />Before downloading the program, please ensure the serial port driver is installed. The “ch341ser.exe” driver file can be found in “Keil Installation Package and mcuisp Downloader”

You can watch “Download Program” demo video in “Sample Program” folder.

1)  Connect USB downloader to controller through Dupont Line. Ensure GND to GND, RXD to TX, and TXD to Rx when connecting.

2)  <img class="common_img" src="../_static/media/chapter_3/section_3/media/image2.png" style="width:800px" />Remove the Jumper cap, framed in red, on STM32 to make STM32 enter burning mode.

<img class="common_img" src="../_static/media/chapter_3/section_3/media/image3.png" style="width:800px" />

3)  Insert the downloader to any interface of the computer, and then switch on the controller. And you need to know the corresponding interface. Right click “Computer” and select “Attribute and Device Manager” in turns.

<img class="inline-icon" src="../_static/media/chapter_3/section_3/media/image4.png" style="width:30px" />If you remove the Jumper cap after switching on the controller, you need to press RST key to enter burning mode.

<img class="common_img" src="../_static/media/chapter_3/section_3/media/image5.jpeg" style="width:800px" />

4)  Mcuisp downloader will also be used. Please open <img class="inline-icon" src="../_static/media/chapter_3/section_3/media/image6.png" style="width:30px" /> saved in “Keil Installation Package and mcuisp Downloader” folder.

5)  Select the corresponding serial port and baud rate. Here take COM11 for example. The interfaces may vary from each computer and please select the corresponding interface. COM1 generally is system communication port not actual port of the controller. The baud rate is 115200, and please don’t modify it.

<img class="common_img" src="../_static/media/chapter_3/section_3/media/image7.jpeg" style="width:800px" />

6)  Click <img class="inline-icon" src="../_static/media/chapter_3/section_3/media/image8.png" style="width:30px" /> in the interface and select the folder where .hex file will be stored.

<img class="common_img" src="../_static/media/chapter_3/section_3/media/image9.jpeg" style="width:800px" />

7)  Next, set the “**STMISP**”, and uncheck “ Execute after Programming” and other options remain the same.

<img class="inline-icon" src="../_static/media/chapter_3/section_3/media/image10.png" style="width:30px" />“Execute after Programming”: the program will be ran as soon as the program is burned into the microcontroller successfully. Hence it is not recommended to check this option. If you have checked this option, you need to keep away from the robotic arm’s moving area to avoid injury.

<img class="common_img" src="../_static/media/chapter_3/section_3/media/image11.jpeg" style="width:800px" />

8)  After setting, click “Start Programming” button, then the program can be burned into the controller. When the message “Anything OK” appears, it means that the program is burned completely.

<img class="common_img" src="../_static/media/chapter_3/section_3/media/image12.jpeg" style="width:800px" />

<img class="inline-icon" src="../_static/media/chapter_3/section_3/media/image13.png" style="width:30px" />If it prompts "chip times out and does not response", it means that it has not switched to burning mode. First check whether the jumper cap is unplugged, and then press the RST button.

9)  Next, we can disconnect the USB downloader, and then reinsert the jumper cap into the STM32. Press the RST button to switch to the running mode. At this time, we can observe that the blue signal on the controller will continue to flash.

<img class="common_img" src="../_static/media/chapter_3/section_3/media/image14.png" style="width:800px" />

<img class="inline-icon" src="../_static/media/chapter_3/section_3/media/image15.png" style="width:30px" />You can refer to this lesson to download the program in “4. Intelligent Games Lesson” and “5. Remote Control”.
